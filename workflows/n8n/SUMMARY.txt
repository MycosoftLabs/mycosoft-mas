WORKFLOWS CREATED
================================================================================

1. myca-discord-to-mas.json (7 nodes)
   - Discord event trigger on "MYCA" mentions or #myca channel
   - Posts to MAS Orchestrator
   - Splits responses >2000 chars for Discord limit compliance
   
2. myca-slack-to-mas.json (5 nodes)
   - Slack app_mention and message.channels triggers
   - Posts to MAS Orchestrator
   - Preserves thread_ts for threaded responses
   
3. myca-asana-to-mas.json (9 nodes)
   - Webhook trigger on task create/update with MYCA tag
   - Smart routing: Morgan (strategic), RJ (operations), Garret (infrastructure)
   - Adds MAS response as task comment
   
4. myca-signal-to-mas.json (5 nodes)
   - Webhook trigger from Signal REST API
   - Posts to MAS Orchestrator
   - Sends plain text reply back to sender
   
5. myca-whatsapp-to-mas.json (5 nodes)
   - Webhook trigger from WhatsApp/Baileys
   - Posts to MAS Orchestrator
   - Sends formatted WhatsApp message reply
   
6. myca-gmail-to-mas.json (7 nodes)
   - Gmail trigger for schedule@mycosoft.org
   - Domain validation: @mycosoft.org only
   - Posts to email registry route
   - Sends professional email reply with MYCA signature
   
7. myca-notion-sync.json (6 nodes)
   - Cron trigger daily at 3 AM UTC
   - Lists Google Drive files modified in last 24 hours
   - Updates Notion database with file records
   - Posts summary to Slack #myca-logs
   
8. myca-doc-query.json (5 nodes)
   - Webhook GET at /webhook/myca/doc-query?q=search_term
   - Searches Notion database
   - Returns JSON response with titles, links, snippets
   
9. myca-health-check.json (9 nodes)
   - Cron trigger every 5 minutes
   - Parallel health checks: n8n, MAS, Signal
   - Alerts to Slack #myca-alerts on failures
   - Logs all results for monitoring
   
10. myca-response-router.json (17 nodes)
    - Webhook POST at /webhook/myca/route-response
    - Platform-aware formatting and routing
    - Error handling: 3 retries with exponential backoff (1s, 2s, 4s)
    - Fallback alert to Slack #myca-alerts

SUPPORT & TROUBLESHOOTING
================================================================================

→ Check service status at health endpoints manually

Issue: Email not received
→ Verify sender domain is @mycosoft.org

Issue: Task comment not updated
→ Check Asana API token validity and task has MYCA tag

Issue: Response never delivered
→ Check #myca-alerts for retry failures

Issue: Webhook timeout
→ Increase timeout in HTTP Request node parameters

Issue: Character limit errors
→ Discord responses auto-split; verify splitting works

Issue: Threading not preserved
→ Verify thread_ts is passed correctly in Slack workflow

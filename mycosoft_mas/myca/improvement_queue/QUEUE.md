# MYCA Improvement Queue

**Date:** February 17, 2026
**Last Updated:** February 17, 2026

## Purpose

This queue tracks proposed improvements to the MYCA system. Items here are generated by:
- Critic agent analyzing Event Ledger failures
- Manual proposals from developers
- Security agent identifying hardening opportunities

## Queue Format

Each item includes:
- **ID**: Unique identifier
- **Type**: skill | policy | permission | eval | code
- **Priority**: P0 (critical) | P1 (high) | P2 (medium) | P3 (low)
- **Status**: proposed | approved | in_progress | pr_ready | merged | rejected
- **Description**: What needs to change
- **Evidence**: Event Ledger IDs, eval failures, etc.
- **Proposed By**: Agent or human
- **PR**: Link to pull request when created

## Process

### 1. Proposal
Critic agent or developer adds entry to this queue with status `proposed`.

### 2. Review
Security agent reviews for safety implications. Manager agent reviews for priority.

### 3. Approval
Human approves the proposal, status changes to `approved`.

### 4. Implementation
Developer agent implements the change, status changes to `in_progress`.

### 5. PR Creation
Pull request is created, status changes to `pr_ready`.

### 6. Merge
After CI passes and human approves, PR is merged, status changes to `merged`.
